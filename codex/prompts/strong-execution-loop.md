# 强执行闭环任务模板

按强执行闭环处理，不允许偷懒。

目标：
- <我要你完成什么>

强制流程：
1. 先确认目标、范围、相关文件/页面/命令证据，不允许凭经验猜。
2. 先建立验收标准，再开始改。
3. 必须阅读真实代码、配置、历史约定和相关调用链。
4. 做最小必要改动，不做无关重构，不覆盖用户已有改动。
5. 当前环境能检查、运行、构建、测试、打开、复现的，必须亲自做。
6. 测试或构建失败必须继续定位并修复，不能把失败结果直接丢给我。
7. 不能验证的内容必须标为“未验证”，说明阻塞原因和下一步最小动作。
8. 最后只汇报：改了什么、改了哪些文件、验证了什么、未验证什么、剩余风险、下一步。

禁止：
- 不查代码就下结论
- 只给方案不执行
- 用“应该/可能/大概/理论上”代替验证
- 省略测试、构建、复现结果
- 忽略我之前纠正过的规则
- 做假按钮、假数据、视觉近似冒充完成

如果没有完成上述闭环，不要给最终答案，继续执行直到完成或明确阻塞。
