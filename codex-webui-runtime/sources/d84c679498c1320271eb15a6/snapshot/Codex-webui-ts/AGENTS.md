# Agent 执行入口

本项目目标是把 Codex-webui-ts 改造成接近 Codex App Plus 的可长期维护 Web UI。所有改动必须先读 Plus 源码和当前项目真实代码，再做最小必要实现，并同步遵守当前项目级 [RULES.md](RULES.md)。

## 强制规则

1. Plus UI/功能还原必须做功能等价，不得只做静态外观。
2. 禁止假按钮、固定提示词、假数据或不可用控件冒充完成。
3. 用户指出截图/UI 异常时，必须复现当前页面状态并检查真实 DOM、接口返回、前端控制台等价证据和相关代码，再修复；禁止只看代码或只凭截图猜测。
4. 会话链路必须真实可用：会话列表、历史点击/恢复、消息展示、发送对话、停止/重启、错误态都要验收。
5. 测试、部署或 UI 验证创建的临时会话必须使用可识别测试标记，验证后删除或从会话列表隐藏，不得留在 lop 的真实对话里；右键删除或清理会话必须按日期移动到 `Codex_RECYCLE` 冷归档并从列表隐藏，不得硬删；删除/隐藏能力未实现时必须列为缺口继续修复或明确阻塞。
6. 读取、排查或修改项目时，必须先按目录结构建立入口索引：根目录、关键配置、项目规则、构建/测试入口和 `src/`、`public/`、`server/` 等主目录，再顺调用链深入。
7. 用户指出目标未对齐、做了没区别、没按目标或“之前不清楚可以先问清楚”时，必须先暂停继续改动，复述目标、列当前差距和验收口径，并只提 1-3 个必须确认的问题；禁止继续按局部问题自作主张。
8. 代码必须保持短而清晰；按职责拆到 `src/`、`public/js/`、`public/css/`，禁止把新逻辑堆成巨型文件。
9. 后端优先复用现有 Node/TS + Codex JSON/SSE 链路，不引入无必要的新框架。
10. `/sessions` 必须使用文件元信息缓存；目标是在约 100 个会话内热路径低于 500ms。
11. “打开文件夹”必须按 Codex App Plus 源码逻辑做功能等价：支持项目目录分级浏览、UI 点击选择项目 cwd、在资源管理器中打开目录三类真实动作；禁止只支持手输路径或把资源管理器打开冒充项目切换；交付前必须反复复查缺少的主功能。
12. 当 lop 指定 Codex WebUI/Plus 基本 UI 与功能还原范围时，必须按从左到右优先级执行：对话按钮功能、对话框显示格式、对话框支持的超链接格式、左侧导航栏显示、对话栏显示、项目分类、设置页、设置选项、其他基本功能；禁止把截图或 Plus 源码里的高级设置项扩成当前任务必做范围。
13. Codex App Plus 全量等价必须按 Plus 源码目录和文件逐段建立清单；除 lop 明确不要和当前已验证等价实现的功能外，必须逐段分析、映射到当前 Node/TS/Web 架构、实现并验证，每项记录来源文件、目标文件、状态和验证证据；禁止跳段、只还原局部主功能、用已实现局部代表整体或用视觉近似冒充源码逻辑转换完成。
14. 当 lop 要求“一次做好、每一段、全量、完全实现、挨个还原、挨个转换”时，验收目标必须是 Plus 全量矩阵清零：开始实现前必须先建立 source-to-target ledger，记录每个 Plus 源文件/片段的来源路径、功能语义、当前状态、目标文件、验收方式和证据；最终交付前必须检索 ledger 和源码中 `missing`、`partial`、`in_progress`、`unmapped`、`unknown`、`todo`、`placeholder`、`fake`、`固定提示词`、`未迁移` 等未完成/伪等价信号，除 lop 明确排除或已有等价验证外，任何命中都禁止声明完成或停止在阶段性交付。`lop-long-task` 的最小可交付切片只适用于 lop 明确允许分期时；在本项目全量等价任务中，完成 P0、第一批或局部主链路只能作为中间进度，必须继续执行到矩阵清零、被 lop 明确叫停，或达到阻塞规则后报告真实阻塞。
15. 当 lop 点名账户、导航栏账户信息、对话框、speed、计划模式、完全访问权限、+ 号菜单、模型/权限/发送按钮等具体 Plus 功能块，或说“这部分一对一/完全还原/按源码逻辑”时，该命名块必须按 Plus 源码逻辑、UI、字段、入口、状态、错误态和可见信息做 source-to-target ledger 与逐项验收；不得用“基本 UI 范围”“高级功能/设置页不在范围”或旧记忆中的 basic scope 排除、降级或转移到设置页。
16. 当 lop 点名 timeline、工具卡、file change/fileChange、diff/patch 卡片或默认展开/折叠/隐藏状态时，默认可见状态、summary 文案、状态徽标、详情内容和 toggle/disclosure 都属于 Plus 源码等价验收；file change/diff/patch 外层详情默认收起，交付前必须断言真实 DOM 状态。
17. Codex App Plus 工程规则已按 WebUI 架构映射到 `RULES.md` 的“Codex App Plus 工程规则映射”章节；涉及目录结构、模块边界、配置、SSE/事件、前后端契约、生成文件、测试或工作区安全时必须读取并执行该章节。

## 验收口径

- `npm run build` 必须通过。
- `npm test` 必须通过。
- 当前 5055 页面必须验证 `/health`、`/sessions`、`/session-messages` 和一次真实 SSE 对话。
- 修 UI 时必须对照截图、Plus 源码、当前页面 DOM/接口证据后再交付。
