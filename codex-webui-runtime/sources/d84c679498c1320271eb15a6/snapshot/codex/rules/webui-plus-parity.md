# Codex WebUI/Plus 源码等价专项规则

读取触发：任务提到 Codex WebUI、Codex-webui-react、React 还原、Web UI 源码、源码迁移、Plus、截图还原、源码等价、导航栏、左侧项目区、项目分类、打开文件夹、对话框、对话框引导、排队/队列、用户消息重编辑、左侧对话栏、相对时间、运行中对话、timeline、工具卡、file change/fileChange、diff/patch 卡片、默认展开/折叠、speed、计划模式、完全访问权限、+ 号菜单、模型/权限/发送按钮，或 lop 说“这部分一对一/完全还原/按源码逻辑”。

## 基础链路

- UI 还原或迁移不能只换静态外观；必须跑通会话列表、历史点击/恢复、消息展示、发送对话、停止/重启、错误态。
- 参考截图/客户端 UI 还原必须逐项对比参考源码/截图的布局、间距、层级、文案、按钮语义和真实功能。
- 禁止用自定义简化按钮、假按钮、假数据、固定提示词或视觉近似冒充原功能。
- 用户指出网页截图/UI 异常时，必须复现当前页面状态，检查真实 DOM、接口返回、前端控制台和相关代码后再修复。

## 范围控制

- lop 提到“最开始、原始需求、原截图、一开始”或指出范围不是全部时，必须暂停扩展实现，回到最早明确截图、文字和验收项重建范围。
- “UI 和基本功能的全量实现”默认指目标 UI 外观、导航、会话/发送/停止/历史/错误等基础链路完整，不等于上游全部高级模块。
- 后续“全量、一次做好、每一段”只在已确认范围内生效，除非 lop 明确要求全仓库、全源码、所有 Plus 模块或所有高级功能。

## 全量/源码等价

- 当 lop 要求“完全实现源码逻辑/UI/功能、一次做好、每一段、全量、挨个还原、挨个转换”时，必须建立 source-to-target ledger。
- ledger 至少记录来源路径/片段、功能语义、当前状态、目标文件、验收方式和证据。
- 最终交付前必须检索 ledger 和源码中的 `missing`、`partial`、`in_progress`、`unmapped`、`unknown`、`todo`、`placeholder`、`fake`、`固定提示词`、`未迁移`、`not implemented` 等信号；除 lop 明确排除或真实阻塞外，任何命中都不能声明完成。
- P0、第一批或主链路通过只能算中间进度，不能替代全量矩阵清零。

## React 源码还原/迁移

- 在 `Codex-webui-react` 或类似 React 重写项目中，lop 提到“React 还原 Web UI 源码、源码迁移、源码等价、完全还原、不要偷懒”时，默认来源是同级 `Codex-webui-ts`；除非 lop 另指来源，不能把任务降级成基础壳、截图近似或功能演示。
- React 迁移 ledger 必须拆到来源函数、DOM 区块、API 调用、状态机、入口动作、空/加载/错误态、移动端或低分辨率表现；禁止用整文件、整模块或“route coverage implemented”一行覆盖多个用户可见行为。
- `Known Parity Notes`、`follow-up polish`、`当前用户 scope`、`不做 pixel-perfect`、`JSON first`、`后续优化` 只能标记未完成、显式排除或阻塞，不能把缺失源码行为包装为 `implemented`。
- 后端文件已复制、接口返回 200、截图能打开或 JSON 面板能显示数据，只能作为辅助证据；必须证明 React 前端有对应入口、交互、状态更新、错误反馈和运行态链路。
- 声称 React 源码还原完成前，必须检索目标源码、source-to-target ledger、测试和验证文档中的 `missing`、`partial`、`in_progress`、`unmapped`、`unknown`、`todo`、`placeholder`、`fake`、`固定提示词`、`未迁移`、`not implemented`、`follow-up`、`polish`、`Known Parity Notes`；规则文件、备份文件和第三方构建产物不计入该禁用词检索。除 lop 明确排除外，任何命中都要继续修复或在最终标为未完成。

## 命名功能块

- lop 点名账户、导航栏、左侧导航栏项目区、项目列表、项目分组、项目切换、项目 cwd、对话框、对话框引导、输入排队/队列、用户消息重编辑、左侧对话栏、对话相对时间、运行中对话常驻显示、timeline、工具卡、file change/fileChange、diff/patch 卡片、默认展开/折叠、speed、计划模式、完全访问权限、+ 号菜单、模型/权限/发送按钮等功能块时，该块必须按 Plus 源码逻辑、UI、字段、入口、状态、错误态和可见信息逐项验收。
- 不得用“基础范围”“高级功能/设置页不在范围”或旧记忆中的 basic scope 排除、降级或转移到设置页；旧 basic scope 只能限制未点名范围，不能降低已点名块验收。

## Timeline 与工具卡状态

- lop 点名 timeline item、工具卡、file change/fileChange、diff/patch 卡片，或要求默认展开、折叠、隐藏时，默认可见状态、summary 文案、状态徽标、详情内容、toggle/disclosure 交互都属于源码等价验收项，不是视觉细节。
- 对话 timeline 中的 file change/diff/patch 外层详情必须按 Plus 源码默认收起；除非 Plus 源码或 lop 明确要求默认展开，禁止把完整 diff/patch 默认展开。
- 交付前必须用静态测试或运行态 DOM 断言 `details.open`、`aria-expanded`、`hidden` 等真实状态；只断言类名、文本或卡片数量不算完成。

## 左侧导航栏项目区

- lop 点名“左侧导航栏项目部分、项目分类、项目列表、项目区、打开文件夹、项目 cwd”任一表达时，必须作为局部源码等价任务。
- ledger 必须覆盖 Plus 源码中的项目数据来源、分组/最近项目/当前项目显示、选择与切换 cwd、项目目录浏览、系统资源管理器打开、空状态、加载态、错误/权限态、折叠/移动端状态、与会话恢复/新建对话联动。
- 当 lop 已要求“按项目分类、项目分类、不需要最近、不要最近、这部分不需要最近、最近项目也不要”任一口径时，左侧顶层不得再展示全局“最近”会话区，项目区内部也不得展示“最近项目”分组或用“最近”命名折叠/空状态；会话入口必须收敛到项目分组内。置顶、运行中、搜索结果等例外必须有独立语义和验收证据，禁止继续命名为“最近”。
- 禁止只实现静态显示、手输路径、假按钮或把资源管理器打开冒充项目切换。
- 交付前必须用源码映射、DOM/API、运行态点击和截图证据逐项验收，并检索当前 UI 结构/DOM/源码中是否仍有未被明确允许的“最近”入口或“最近项目”文案；任何子项未覆盖或仍保留禁用入口，都必须标为未完成或阻塞。

## 对话与左侧对话栏细节

- lop 点名对话框引导、排队/队列、用户消息重编辑、左侧对话栏时间、几分钟/小时前或运行中对话显示时，必须作为局部源码等价任务。
- ledger 必须覆盖 Plus 源码中的输入框引导/空状态、发送中与排队状态、队列取消/继续/失败态、用户消息 hover/菜单/编辑入口、编辑草稿回填、编辑提交后从该消息继续或重跑的真实链路、会话列表相对时间计算与刷新、刚刚/N 分钟前/N 小时前/跨天日期显示、运行中会话常驻可见/置顶或排序/状态徽标/流式更新、历史恢复和新建对话联动。
- 禁止只做静态文案、假排队、不可点击编辑、固定时间、刷新后运行中会话消失或用视觉近似冒充完成。
- 交付前必须用 Plus 源码映射、DOM/API、运行态点击、可控时间或日志证据逐项验收；任何子项未覆盖都必须标为未完成或阻塞。
